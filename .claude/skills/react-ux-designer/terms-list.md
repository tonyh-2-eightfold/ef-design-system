# Terms List — Approved Eightfold Product Vocabulary

Canonical terminology for all UI copy, labels, headings, and feature names in Eightfold-built products. Always use the **Approved** form. Do not use the **Avoid** alternatives even if they appear more natural — consistency across the product drives user trust and recognition.

---

## How to Use This List

1. Before writing any screen copy, check the term you're about to use against the **Approved** column.
2. If a term appears in the **Avoid** column, replace it with the approved form.
3. When in doubt, prefer shorter, plainer English over technical or corporate vocabulary.
4. These terms apply to all UI surfaces: labels, headings, button text, tooltips, empty states, notifications, and error messages.

## UI and Interaction Terms

| Approved | Avoid | Notes |
|----------|-------|-------|
| Search | Find, look up, query | |
| Filter | Narrow, refine, sort (when meaning filter) | "Sort" is correct only for ordering |
| Sort | Order, arrange, filter (when meaning order) | Keep sort and filter conceptually distinct in UI |
| Save | Submit, confirm, apply (for persistent state changes) | "Apply" is acceptable for filter application |
| Cancel | Close (for destructive confirmation dialogs), No | Close is for closing panels/modals, Cancel is for abandoning an action |
| Close | Dismiss, exit, X (in labels) | The × icon action label should be "Close [context]" |
| Remove | Delete (for non-permanent associations), unlink | Use "Remove" when the item still exists elsewhere; "Delete" when it's permanent |
| Delete | Remove (for permanent destruction) | "Delete" implies permanence — use only when the action is irreversible |
| Edit | Modify, update, change (in button labels) | "Update" acceptable in success messages: "Profile updated." |
| Add | Create, new (in button labels when adding to a collection) | "New [item]" acceptable for creation of standalone entities |
| View | See, open, show (in link/button labels) | |
| Export | Download, extract, pull | "Download" acceptable when referring to a file the user receives |
| Share | Send, distribute | |
| Archive | Deactivate, hide, soft-delete | Reserve for the specific archive action; don't use as synonym for Delete |
| Invite | Add (when adding external users via email) | |
| Assign | Allocate, give, delegate | |

## Technical Terms (Use Plainly)

These technical concepts appear in the product and must be explained in plain language when shown to non-technical users.

| Term | Plain language equivalent | When to use which |
|------|--------------------------|-------------------|
| Algorithm / AI model | "Eightfold's matching technology" or "AI" | Use "AI" for user-facing copy; technical detail for admin/settings only |
| Data ingestion | "Importing your data" | Use plain language for all non-admin users |
| API integration | "Connect [system name]" | Button/CTA language; technical term in developer docs only |
| SSO | "Single sign-on" or "Sign in with [provider]" | Spell out in onboarding; SSO acceptable in IT admin settings |
| ATS | "Your applicant tracking system" or name the system | Spell out unless the user is clearly technical |
| HRIS | "Your HR system" or name the system | Spell out unless the user is clearly technical |
| JD | "Job description" | Spell out in UI; "JD" acceptable in table headers |

## Numbers, Dates, and Quantities

| Context | Format | Example |
|---------|--------|---------|
| Counts under 10 | Spelled out | "three candidates" (in prose) |
| Counts 10+ | Numerals | "24 candidates" |
| Counts in data UI | Always numerals | "3 skills", "1 role" |
| Large numbers | K/M suffix | "1.2K", "23M" |
| Percentages | Numeral + % | "87% match" |
| Date in prose | Month D, YYYY | "March 5, 2025" |
| Date in tables | MM/DD/YYYY or YYYY-MM-DD (per locale) | "03/05/2025" |
| Relative dates (recent) | "X days ago", "Today", "Yesterday" | Under 30 days |
| Relative dates (older) | Full date | "Jan 15, 2024" |
| Time | 12-hour with AM/PM | "2:30 PM" |
| Duration | "X hours" / "X days" | "2 hours", "5 days" — no abbreviations in prose |

## Inclusive Language

| Avoid | Use Instead | Notes |
|-------|-------------|-------|
| Blacklist / whitelist | Block list / allow list | |
| Master / slave | Primary / secondary, leader / follower | |
| Manpower | Workforce, talent, staff | |
| Guys (for mixed groups) | Team, everyone, folks | |
| He/she (generic) | They / their | Singular they is standard |
| Handicapped | Disabled, person with a disability | Person-first unless individual prefers identity-first |
| Minority | Underrepresented group | More specific and accurate |
| Hire the best person for the job | Hire the most qualified candidate | Removes implicit bias framing |
